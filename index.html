Introduction

CλaSH (pronounced ‘clash’) is a functional hardware description language that borrows both its syntax and semantics from the functional programming language Haskell. The merits of using a functional language to describe hardware comes from the fact that combinational circuits can be directly modeled as mathematical functions and that functional languages lend themselves very well at describing and (de-)composing mathematical functions.

An experimental version of the new CλaSH compiler and some examples can be found below.
!! The compiler and the CλaSH Prelude library are subject to change !!

Installation

Because the constraint solver for type-level naturals is only included in the upcoming 7.8 release of GHC, installation of CλaSH is sadly more involved than a simple cabal install clash. The instructions for the installation are found below; these instructions assume that no previous versions of GHC are present on the system. If you want to run CλaSH alongside an existing installation of GHC, you should investigate solutions such as stow or multi-ghc.

  1. Install GHC 7.8-rc2
    • Download GHC 7.8-rc2 for your platform (See GHC 7.8-rc2)
    • Unpack (tar xf) the tarball and cd to the directory
    • Run ./configure --prefix=<INSTALLDIR>, where <INSTALLDIR> is the directory you want to install GHC
    • Run make install
    • Add the bin directory of GHC to your path: PATH=<INSTALLDIR>/bin:$PATH
  2. Install Cabal
    • Download the sources for cabal (See Cabal)
    • Unpack (tar xf) the archive and cd to the directory
    • Run: sh bootstrap.sh
    • Follow the instruction to add the cabal program to your PATH
    • Run: cabal update
  3. Install the modified RepLib library
    • Download the sources for the RepLib library (See Modified Libraries)
    • Run cabal install RepLib-0.5.3.1.tar.gz
  4. Install CλaSH
    • Run cabal install clash-ghc
  5. Verify everything is working by:
    • Downloading the FIR.hs example
    • Run: clash --interactive FIR.hs
    • Executing the :vhdl command, leave the interpreter using :q
    • Examine the VHDL code in the vhdl directory
