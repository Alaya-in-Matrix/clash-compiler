CλaSH

CAES Language for Synchronous Hardware

Introduction

CλaSH (pronounced ‘clash’) is a functional hardware description language that borrows both its syntax and semantics from the functional programming language Haskell. The merits of using a functional language to describe hardware comes from the fact that combinational circuits can be directly modeled as mathematical functions and that functional languages lend themselves very well at describing and (de-)composing mathematical functions.

An experimental version of the new CλaSH compiler and some examples can be found below.
!! The compiler and the CλaSH Prelude library are subject to change !!

Installation

Because the type-nats branch is not merged into mainline GHC, installation of CλaSH is sadly more involved than a simple cabal install clash. The instructions for the installation are found below; these instructions assume that no previous versions of GHC are present on the system. If you want to run CλaSH alongside an existing installation of GHC, you should investigate solutions such as stow or multi-ghc.

  1. Install GHC-HEAD
    • Download GHC-HEAD for your platform (See GHC-HEAD (type-nats branch))
    • Unpack (tar xjf) the tarball and cd to the directory
    • Run ./configure --prefix=<INSTALLDIR>, where <INSTALLDIR> is the directory you want to install GHC
    • Add the bin directory of GHC to your path: PATH=<INSTALLDIR>/bin:$PATH
  2. Install the CλaSH binary
    • Download the CλaSH binary for your platform (See CλaSH Binaries) to $INSTALLDIR/bin and rename it to clash
  3. Install the TH-Lift library
    • Download the TH-Lift library (See TH-Lift package)
    • Unpack (tar xzf / unzip) the archive and cd to the directory
    • Run: runhaskell Setup.hs configure --global --prefix=<INSTALLDIR>, where <INSTALLDIR> is the location you installed GHC
    • Run: runhaskell Setup.hs build
    • Run: runhaskell Setup.hs install
  4. Install the CλaSH prelude library
    • Download the CλaSH prelude (See CλaSH Prelude)
    • Unpack (tar xzf / unzip) the archive and cd to the directory
    • Run: runhaskell Setup.hs configure --global --prefix=<INSTALLDIR>, where <INSTALLDIR> is the location you installed GHC
    • Run: runhaskell Setup.hs build
    • Run: runhaskell Setup.hs install
  5. Install the CλaSH primitive definitions
    • Download the CλaSH primitives (See CλaSH Primitive Definitions)
    • Unpack (tar xzf) the primitives tarball to <PRIMDIR>, where <PRIMDIR> is the directory where you want to store the primitive definitions
    • Add a new environment variable (CLASH_PRIMDIR) pointing to the <PRIMDIR> directory: export CLASH_PRIMDIR="<PRIMDIR>"
  6. Verify everything is working by:
    • Downloading the FIR.hs example
    • Run: clash --interactive FIR.hs
    • Executing the :vhdl command, leave the interpreter using :q
    • Examine the VHDL code in the vhdl directory
